Design Notes

  • ‚úÖ Auto-run workflow: No "Run Algorithm" button - runs automatically when config changes (fast!)
  • ‚úÖ Conceptual order: Configuration ‚Üí Playback ‚Üí Metrics matches the mental model (setup, run, observe)
  • ‚úÖ Matches existing styles: Uses your Tailwind classes (bg-white, rounded-lg, shadow-md, text-gray-900, etc.)
  • ‚úÖ Clear interaction model: "EDITABLE" badges on config, "NAVIGATION" on playback, "READ ONLY" on metrics
  • ‚úÖ Video player metaphor: Timeline scrubber looks familiar, Previous/Next like media controls
  • ‚úÖ Convergence-first hierarchy: Gradient norm prominent with progress bar in hero section
  • ‚úÖ Rich detail from compact dashboard: Includes gradient decomposition, direction vectors, eigenvalues, condition number
  • ‚úÖ Mathematical precision: Shows ||‚àáf||‚ÇÇ, max component, ||Œîw||‚ÇÇ, Œ∫ (condition number), Œª‚ÇÅ/Œª‚ÇÇ (eigenvalues)
  • ‚úÖ Line search visualization: Plot shows actual loss curve, Armijo bound, and trial points (like current UI, but integrated)
  • ‚úÖ Conditional sections: Line search panel only for GD-LS/Newton/L-BFGS; Hessian panel only for Newton
  • ‚úÖ Pluggable line search: UI notes "Armijo (pluggable)" to indicate extensibility
  • ‚úÖ Consistent spacing: Uses existing p-6, p-4, mb-6, mb-4 patterns
  • ‚úÖ Existing color palette: blue-600 for primary actions, gray-50 for info boxes, amber-50 for convergence, blue-50 for line search, purple-50 for Hessian
  • ‚úÖ Trend indicators: Shows changes (‚Üì -12.424, -16.0%) and reduction rates for loss and gradient

What's Different from Current UI:

  • 1. No "Run" button: Auto-runs on config change (settings you see = what ran, no mismatched state)
  • 2. Clear separation: Config (editable) at top, playback (navigation) in middle, metrics (read-only) at bottom
  • 3. Video player controls: Timeline scrubber makes navigation intent obvious
  • 4. Convergence hero: Gradient norm gets prominent placement with visual progress bar
  • 5. Richer metrics: Shows gradient components, direction, movement magnitude, reduction rates
  • 6. Integrated visualizations: Line search plot and Hessian info moved from bottom to iteration section
  • 7. Algorithm-specific panels: Line search (blue) and Hessian (purple) only shown when applicable

Conditional Display Logic:

  • GD Fixed Step: No line search panel, no Hessian panel (simplest view)
  • GD Line Search: Show line search panel (blue), no Hessian panel
  • Newton's Method: Show line search panel (blue) + Hessian panel (purple)
  • L-BFGS: Show line search panel (blue), no Hessian panel (quasi-Newton uses approximation)
  • Future line search methods: Pluggable - just update the visualization data source
