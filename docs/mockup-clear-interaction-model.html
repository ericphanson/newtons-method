Design Principles

1. Clear Visual Separation

  • Navigation (Dark gray): Media player controls for stepping through results
  • Observation (Light gray): Read-only display of iteration metrics
  • Configuration (White with blue borders): Editable inputs that trigger re-computation

2. Interaction Affordances

  • Timeline scrubber: Looks like video player (YouTube, etc.) - clearly for navigation
  • Play button: Auto-advance through iterations (animation)
  • Configuration sliders: Different visual style (blue) - clearly for input
  • "Run Algorithm" button: Explicit trigger for re-computation

3. Status Badges

  • "READ ONLY" badge: Makes it clear metrics can't be edited directly
  • "EDITABLE" badge: Signals this is the configuration area
  • "EDIT" micro-badges: On each input for extra clarity

4. Navigation Placement

  • Top position: Navigation controls at top (like video player header)
  • Visually distinct: Dark background separates from content
  • Large iteration counter: Current position always visible
  • Timeline scrubber: Full width for precise control

Key Insight: Mental Model Alignment

This design maps to a familiar mental model: video player. Users understand that:

  • Playback controls navigate through pre-recorded content (don't change the video)
  • Settings (quality, speed) modify the experience (trigger new behavior)
  • The timeline shows where you are in the recording

Same principle here: Navigation steps through pre-computed iterations, Configuration changes the algorithm and re-runs.
